Git

Git is a version control system

Git allows teams to work together

git init

Run command above to initialize a repository

Introduction

It works with other technologies such as github and bit bucket

Git is open source, meaning it's not owned by anyone.

git add {param}

Run command above to add a file/ folder for git to track

Branch

Git uses branches to separate each users work

Normally there is a master branch which acts as the main branch

git checkout params

Run command above to change branches

Merge

The concept of merging allows users to combine different branches and thus their work.

Userscan go back on merged branches through git's history

git rebase [options, params]

Run command above to rebase from another branch

git push [options, params]

Finally push to send changes to remote repository

Advantages

  1. Saves developers a lot time
  2. Allows more hands on to one project
  3. Allows users to track project history
  4. Keeps track of user contributions
  5. Allows management of application verstiosn

Team collaboration is made a whole lot easier by git

Life without git would be a whole lot hard
